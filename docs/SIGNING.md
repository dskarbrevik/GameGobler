# Code Signing & Auto-Updater Setup

The release workflow (`.github/workflows/release.yml`) builds signed native installers via Tauri. This guide explains how to set up the required GitHub repository secrets.

## Tauri Updater Signing (Required)

The auto-updater needs a signing key to verify downloads. One was already generated during setup.

| Secret | Value |
|--------|-------|
| `TAURI_SIGNING_PRIVATE_KEY` | Contents of `tauri-updater.key` (the file generated by `npx tauri signer generate`) |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | Password you entered during key generation |

If you lost the key, generate a new one:

```bash
cd web && npx tauri signer generate -w ../tauri-updater.key
```

Then update the public key in `web/src-tauri/tauri.conf.json` under `plugins.updater.pubkey`.

## macOS Code Signing & Notarization (Optional)

Without these, macOS users will see Gatekeeper warnings ("unidentified developer"). Requires an [Apple Developer Program](https://developer.apple.com/programs/) membership ($99/year).

### Steps

1. **Create a Developer ID Application certificate** in Xcode or the Apple Developer portal.
2. **Export the certificate** as a `.p12` file from Keychain Access.
3. **Base64-encode it:**
   ```bash
   base64 -i Certificates.p12 | pbcopy
   ```
4. **Create an app-specific password** at [appleid.apple.com](https://appleid.apple.com/account/manage) â†’ Sign-In and Security â†’ App-Specific Passwords.

### Secrets

| Secret | Value |
|--------|-------|
| `APPLE_CERTIFICATE` | Base64-encoded `.p12` file contents |
| `APPLE_CERTIFICATE_PASSWORD` | Password for the `.p12` |
| `APPLE_SIGNING_IDENTITY` | `Developer ID Application: Your Name (TEAMID)` â€” shown in Keychain Access |
| `APPLE_ID` | Your Apple ID email |
| `APPLE_PASSWORD` | App-specific password (not your Apple ID password) |
| `APPLE_TEAM_ID` | 10-character Team ID from [developer.apple.com/account](https://developer.apple.com/account) |

## Windows Code Signing (Optional)

Without this, Windows users will see SmartScreen warnings. Requires a code signing certificate from a Certificate Authority (e.g., DigiCert, Sectigo, SSL.com).

### Steps

1. **Purchase an Authenticode code signing certificate** (EV or OV).
2. **Export as `.pfx`** with a password.
3. **Base64-encode it:**
   ```bash
   base64 -i certificate.pfx | pbcopy   # macOS
   certutil -encode certificate.pfx encoded.txt  # Windows
   ```

### Secrets

| Secret | Value |
|--------|-------|
| `WINDOWS_CERTIFICATE` | Base64-encoded `.pfx` file contents |
| `WINDOWS_CERTIFICATE_PASSWORD` | Password for the `.pfx` |

## Linux

Linux `.AppImage` and `.deb` packages don't require code signing. They will be built and uploaded automatically.

## Triggering a Release

Push a version tag to build and publish:

```bash
git tag v2.0.0
git push origin v2.0.0
```

The workflow will:
1. Build the Python sidecar via PyInstaller (Linux, macOS, Windows)
2. Bundle everything into native installers via Tauri
3. Create a GitHub Release with `.dmg`, `.msi`, `.AppImage`, `.deb`, and updater artifacts
